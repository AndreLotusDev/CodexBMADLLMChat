# Story 2.1: Connection Screen & Credential Input

## Status

Done

## Story

**As a** user,
**I want** to enter my PostgreSQL connection details in a form,
**so that** I can connect to my database.

## Acceptance Criteria

1. Connection screen renders a form with fields: Host, Port (default 5432), Database, Username, Password
2. Form validates that all required fields are filled before allowing submission
3. A "Test Connection" button attempts to connect and shows a success or error message inline
4. Connection errors display a human-readable message (e.g., "Could not reach host" vs. "Authentication failed")
5. Password field is masked by default with a show/hide toggle

### Architecture: AppError Enum (Rust)

[Source: architecture/11-backend-architecture.md#115-error-handling]

```rust
use thiserror::Error;
use serde::Serialize;

#[derive(Debug, Error, Serialize)]
#[serde(tag = "code", content = "message")]
pub enum AppError {
    #[error("Could not reach host. Check the hostname and port.")]
    HostUnreachable(String),
    #[error("Authentication failed. Check your username and password.")]
    AuthFailed(String),
    #[error("Database not found. Check the database name.")]
    DatabaseNotFound(String),
    #[error("Connection timed out after {0} seconds.")]
    ConnectionTimeout(u64),
    #[error("Internal error: {0}")]
    Internal(String),
}

impl tauri::ipc::IntoIpcResponse for AppError { ... }
// OR keep the From<AppError> for String impl вЂ” verify which Tauri v2 requires
```

**Tauri v2 error serialization note**: Tauri v2 requires command errors to implement `serde::Serialize`. The `#[serde(tag = "code", content = "message")]` attribute on the enum causes it to serialize as `{ "code": "AuthFailed", "message": "..." }` which matches the `TauriCommandError` TS interface. This is the standard pattern.

### Architecture: ConnectionManager (Rust)

[Source: architecture/11-backend-architecture.md#111-service-architecture, architecture/3-tech-stack.md вЂ” sqlx 0.7.x]

```rust
// src-tauri/src/services/connection_manager.rs
use sqlx::postgres::PgPoolOptions;
use crate::errors::AppError;
use crate::models::TestConnectionParams;

pub struct ConnectionManager;

impl ConnectionManager {
    pub fn new() -> Self { Self }

    pub async fn test(&self, params: &TestConnectionParams) -> Result<(), AppError> {
        let url = format!(
            "postgresql://{}:{}@{}:{}/{}",
            params.username, params.password, params.host, params.port, params.database
        );
        let pool = PgPoolOptions::new()
            .acquire_timeout(std::time::Duration::from_secs(5))
            .max_connections(1)
            .connect(&url)
            .await
            .map_err(|e| map_sqlx_error(e))?;
        pool.close().await;
        Ok(())
    }
}

    // sqlx doesn't expose typed DB error codes cleanly вЂ” match on error string/kind
```

**sqlx error mapping guide:**
- `sqlx::Error::PoolTimedOut` or connection refused в†’ `AppError::HostUnreachable`
- PG error code `28P01` (invalid_password) or `28000` в†’ `AppError::AuthFailed`
- PG error code `3D000` (invalid_catalog_name) в†’ `AppError::DatabaseNotFound`
- `sqlx::Error::PoolTimedOut` в†’ `AppError::ConnectionTimeout(5)`
- Everything else в†’ `AppError::Internal(e.to_string())`

Check sqlx's `PgDatabaseError::code()` for PostgreSQL error codes.

### Architecture: Coding Standards Critical Rules

[Source: architecture/17-coding-standards.md]

- **No password in store**: Password stays in `ConnectionForm` local React state only. Never written to Zustand.
- **IPC Gateway**: All `invoke()` through `src/commands/index.ts`. ConnectionForm imports `commands`, not `invoke` directly.
- **Type mirroring**: Every Rust IPC struct has a matching TS interface in `src/types/index.ts`. `TestConnectionParams` in Rust в†’ same-named type in TS (via `serde rename_all = "camelCase"`).
- **Rust error propagation**: All fallible functions return `Result<T, AppError>`. No `.unwrap()` or `.expect()` in non-test code.
- **Command handlers thin**: `test_connection` handler в‰¤ 15 lines, delegates to `ConnectionManager`.
- **Drop sensitive strings**: After `test()` returns, the password string goes out of scope automatically. No explicit drop needed since it's a by-value function arg.
