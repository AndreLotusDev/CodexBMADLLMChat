# Story 1.1: Initialize Tauri + React Project Scaffold

## Status

Draft

## Story

**As a** developer,
**I want** a Tauri v2 + React (TypeScript) project initialized with the correct folder structure and dependencies,
**so that** the team has a working base to build features on.

## Acceptance Criteria

1. `npm create tauri-app` (or equivalent) generates a working Tauri v2 + React + TypeScript project
2. App launches on Windows with a blank/placeholder window
3. `npm run dev` starts the development server with hot reload
4. `npm run build` produces a `.exe` installer for Windows
5. Project includes ESLint + Prettier configured with sensible defaults
6. Git repository initialized with `.gitignore` covering `node_modules`, `dist`, and Tauri build artifacts

### Vitest Configuration
[Source: architecture/16-testing-strategy.md#16.6]

Add to `vite.config.ts`:
```typescript
test: {
  globals: true,
  environment: 'jsdom',
  setupFiles: ['./src/__tests__/setup.ts'],
  mockReset: true,
}
```

### Tauri IPC Mock for Tests
[Source: architecture/16-testing-strategy.md#16.6]

`src/__tests__/setup.ts`:
```typescript
vi.mock('@tauri-apps/api/core', () => ({ invoke: vi.fn() }))
vi.mock('@tauri-apps/api/event', () => ({ listen: vi.fn(() => Promise.resolve(() => {})) }))
```

### Critical Coding Rules to Enforce from Day One
[Source: architecture/17-coding-standards.md]

- **IPC Gateway:** All `invoke()` calls must go through `src/commands/index.ts` — no component may import `@tauri-apps/api` directly.
- **No Password in Store:** Zustand store must never hold a password field.
- **Rust Error Propagation:** All fallible Rust functions return `Result<T, AppError>`; never `.unwrap()` or `.expect()` in non-test code.
- **SQLite Pragma on Open:** Every SQLite connection opened via `db::open_db()` — never call `Connection::open()` directly.
- **Command Handlers Stay Thin:** `#[tauri::command]` functions ≤ ~15 lines; no business logic in handlers.

### Naming Conventions
[Source: architecture/17-coding-standards.md#17.2]

| Element | Convention | Example |
|---------|-----------|---------|
| React Components | PascalCase | `TableNode.tsx` |
| Hooks | camelCase + `use` prefix | `useDebounce.ts` |
| Store actions | camelCase verbs | `toggleTable` |
| IPC commands (TS) | camelCase | `commands.upsertAnnotation()` |
| IPC commands (Rust) | snake_case | `upsert_annotation` |
| TypeScript types | PascalCase | `ConnectionProfile` |
| Rust structs | PascalCase | `AnnotationRepository` |
| SQLite tables | snake_case | `connection_profiles` |

### Testing

**Frontend:**
- Framework: Vitest 1.x + jsdom
- Test file location: `src/__tests__/`
- All Tauri IPC calls (`invoke`, `listen`) must be mocked in `src/__tests__/setup.ts`
- No E2E tests in MVP

**Backend:**
- Framework: `cargo test` (Rust built-in)
- Unit tests: `#[cfg(test)]` modules within each service/repository file
- Integration tests: `src-tauri/tests/integration/` with `#[ignore = "requires local PostgreSQL"]`
- No integration tests required for this story (no DB logic yet)
