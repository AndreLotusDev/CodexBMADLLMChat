## Story

1. `npm create tauri-app` (or equivalent) generates a working Tauri v2 + React + TypeScript project

4. `npm run build` produces a `.exe` installer for Windows
5. Project includes ESLint + Prettier configured with sensible defaults
6. Git repository initialized with `.gitignore` covering `node_modules`, `dist`, and Tauri build artifacts

### IPC Gateway Rule (Critical Coding Standard)

[Source: architecture/17-coding-standards.md#171-critical-fullstack-rules]

ALL `invoke()` calls MUST go through `src/commands/index.ts`. No component should ever import `@tauri-apps/api` directly. The stub created in this story establishes that gateway file for future commands.

### Rust Error Propagation Rule

[Source: architecture/17-coding-standards.md#171-critical-fullstack-rules]

All fallible Rust functions return `Result<T, AppError>`. Never use `.unwrap()` or `.expect()` in non-test code. The `errors.rs` stub should define the `AppError` enum (even if empty for now).

### Production Build Output

[Source: architecture/14-deployment-architecture.md#141-deployment-strategy]

Build output: `src-tauri/target/release/bundle/nsis/SchemaLift_x.x.x_x64-setup.exe`

MVP ships unsigned вЂ” Windows SmartScreen will warn on first run. This is expected and acceptable for MVP.

## Testing

[Source: architecture/16-testing-strategy.md]

**Frontend Testing:**
- Framework: Vitest 1.x + jsdom
- Test file location: `src/__tests__/`
- Setup file: `src/__tests__/setup.ts` вЂ” must mock all Tauri IPC at module level:
  ```typescript
  vi.mock('@tauri-apps/api/core', () => ({ invoke: vi.fn() }))
  vi.mock('@tauri-apps/api/event', () => ({ listen: vi.fn(() => Promise.resolve(() => {})) }))
  ```
- Vitest config in `vite.config.ts`:
  ```typescript
  test: {
    globals: true,
    environment: 'jsdom',
    setupFiles: ['./src/__tests__/setup.ts'],
    mockReset: true,
  }
  ```
- For this story, no component tests are required вЂ” just confirm Vitest boots with 0 failing tests.

**Backend Testing:**
- Framework: `cargo test` (Rust built-in)
- Test modules: `#[cfg(test)]` inline in each file
- For this story, stub files have no logic to test вЂ” just confirm `cargo test` runs without errors.

**E2E:** None for MVP вЂ” explicitly out of scope.
