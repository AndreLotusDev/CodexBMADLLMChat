# Story 2.3: Schema Search & Filter

## Status

Done

## Story

**As a** user,
**I want** to search and filter the schema tree by name,
**so that** I can quickly find tables or columns in large databases.

## Acceptance Criteria

1. A search input above the tree filters tables and columns in real time as the user types
2. Matching nodes are highlighted; non-matching nodes are hidden
3. Clearing the search restores the full tree
4. Search is case-insensitive

- [x] Task 2: Create `src/hooks/useDebounce.ts` — generic debounce hook (AC: 1)
  - [x] Implement callback-debouncing signature (matches architecture §17.4 example):
    ```typescript
    import { useCallback, useEffect, useRef } from 'react'

    export function useDebounce<TArgs extends unknown[]>(
      fn: (...args: TArgs) => void,
      delayMs: number,
    ): (...args: TArgs) => void {
      const fnRef = useRef(fn)
      const timerRef = useRef<ReturnType<typeof setTimeout> | null>(null)

      useEffect(() => { fnRef.current = fn }, [fn])
      useEffect(() => () => { if (timerRef.current) clearTimeout(timerRef.current) }, [])

      return useCallback((...args: TArgs) => {
        if (timerRef.current) clearTimeout(timerRef.current)
        timerRef.current = setTimeout(() => fnRef.current(...args), delayMs)
      }, [delayMs])
    }
    ```
  - [x] No new npm dependency required — uses only React primitives

- [x] Task 3: Create `src/lib/filterSchemaTree.ts` — pure filter utility (AC: 1, 2, 3, 4)
  - [x] Export `filterSchemaTree(tree: SchemaTree, query: string): SchemaTree`
  - [x] Behavior:
    - If `query.trim() === ''` → return `tree` unchanged (referential equality preferred)
    - Lowercase the query once: `const q = query.trim().toLowerCase()`
    - For each schema, for each table:

### Filter Algorithm & Data Model Reminder

[Source: architecture/4-data-models.md#42-schematree-runtime]

`SchemaTree` shape (already in `src/types/index.ts` from Story 2.1):

```typescript
interface SchemaTree   { schemas: PgSchema[] }
interface PgSchema     { name: string; tables: PgTable[] }
interface PgTable      { schemaName, name, columns: PgColumn[], primaryKeys, foreignKeys }
interface PgColumn     { name, dataType, isNullable, isPrimaryKey, foreignKeyRef? }
```

`filterSchemaTree` rules (Task 3):

1. Empty/whitespace query → return original tree (no work)
2. Table-name match (case-insensitive substring) → keep that table with **all** original columns
3. Column-name match on at least one column in a table → keep the table with **only** the matching columns
4. Schema with no surviving tables after the above → omit the schema
5. Pure function — never mutate input arrays/objects; allocate new ones

Match criteria are substring `.includes()` after lowercasing both sides. Glob/regex patterns are explicitly out of scope (PRD AC #1/#4 say "search and filter by name", "case-insensitive" — substring is the agreed semantic).
